# Code of Conduct

## Our Standard

`Sane` should be a project where people can participate without being talked down to, harassed, or treated as disposable.

We expect contributors and maintainers to:

- be respectful and direct
- assume good faith before escalating
- critique ideas and code, not people
- keep disagreement technical and specific
- make room for beginners and first-time contributors

## Unacceptable Behavior

This includes, but is not limited to:

- harassment or intimidation
- personal attacks
- hate speech or discriminatory language
- sexualized language or unwanted attention
- doxxing or sharing private information
- bad-faith pile-ons, trolling, or repeated disruption

## Enforcement

Maintainers may remove, edit, or reject comments, issues, pull requests, and other contributions that violate this code of conduct.

Serious or repeated violations may lead to temporary or permanent bans from the project spaces.

## Reporting

If you experience or witness unacceptable behavior, contact the maintainer through the repository owner's GitHub profile contact method.

If the report is about public behavior in the repository, include:

- links
- screenshots if relevant
- a short description of what happened

Please do not open a public issue for code-of-conduct reports.

## Scope

This code of conduct applies to:

- GitHub issues
- pull requests
- discussions
- review comments
- other official project spaces
