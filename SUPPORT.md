# Support

## Where to Go

### Bug reports

Open a GitHub issue using the bug template.

### Feature requests

Open a GitHub issue using the feature request template.

### Security issues

Follow [SECURITY.md](./SECURITY.md).
Do not open a public issue for vulnerabilities.

### Documentation problems

Open a normal issue or a pull request.
Docs fixes are welcome.

## Before Opening an Issue

Please include:

- what you expected
- what actually happened
- your OS
- the command or TUI flow you used
- your current commit hash (`git rev-parse HEAD`) if you are running from source
- any logs or screenshots that help reproduce the problem

It also helps to say which Sane surface failed:

- local `.sane` runtime
- `sane-router` skill
- pack skill export
- global `AGENTS.md` block
- hooks
- custom agents
- Codex config profile preview or apply

For install or config problems, details matter.

## Scope Reminder

`Sane` is still pre-release.
If something feels rough around the edges, that may be expected, but reproducible reports are still useful.
